Section 6
Reply Detection & Classification
Runs ALL sweeps (morning, midday, evening). Thread-based: reads sdr_gmail_thread_id for messages FROM the contact's email.
TypeSignal PatternsStatusOutcomehs_lead_status
Positive "interested", "tell me more", "let's set up a call", service questions completed_reply reply_positive CONNECTED
Negative "not interested", "remove me", "unsubscribe", "no thanks" completed_reply reply_negative UNQUALIFIED
OOO "out of office", "back on {date}", auto-reply headers paused — unchanged
Ambiguous No clear match unchanged — unchanged
Ambiguous replies NEVER auto-classified. Always escalated to human via #klaw-rev-output.
Phase B (reply detection) runs BEFORE Phase D (follow-up sends) every sweep — prevents sending after reply.

Section 8
Edge Cases
Reply Between Touches
Phase B (reply check) runs before Phase D (sends). Additional safety: Phase D re-reads Gmail thread before each send.
Duplicate Form Submit
Active seq: bump engagement +5, flag AE. No-reply complete: reset, restart Touch 1. Converted: skip SDR, flag AE.
Already in Pipeline
Pre-check #5 catches. Contact with active deal → disqualified. Deal owner notified of form submission.
Clay Fails
Before T1: blocks start (Safeguard #2), retries next sweep. >48hrs → escalation. Before T2/T3: no dependency, uses stored data.
Score Changes Mid-Sequence
Sequence runs to completion — decision locked via sdr_qualification_score. Exception: score drops to 0 → human review.
Multi-Contact Same Company
Pre-check #7 blocks parallel sequence. New contact enriched/scored. Original gets multi-stakeholder signal, engagement +5.
Unsubscribe Mid-Sequence
Immediate stop. Status = completed_reply, outcome = reply_negative. No nurture pool. Opt-out is permanent.
Nurture Exit (No Reply × 3)
hs_lead_status = BAD_TIMING. Monthly AE task (LOW). 3 months no action → archive. WF5 eligible at 90 days.
